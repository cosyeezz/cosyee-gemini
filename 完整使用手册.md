# Gemini代理负载均衡器 - 完整使用手册

## 📖 项目概述

这是一个用Go语言开发的轻量级Gemini API负载均衡器，专为1核1G小内存服务器优化。相比Python版本，内存占用减少90%+，启动速度提升10倍，部署极其简单。

### 核心特性
- ✅ **超轻量**: 内存占用仅5-10MB
- ✅ **零依赖**: 只使用Go标准库
- ✅ **简单轮询**: Round Robin负载均衡算法
- ✅ **智能健康检查**: HTTP探测后端服务状态
- ✅ **配置驱动**: JSON格式配置文件
- ✅ **详细日志**: 彩色emoji日志，便于监控
- ✅ **单二进制**: 无依赖部署
- ✅ **故障自愈**: 自动检测和恢复失效API密钥

## 🎯 适用场景

- 1核1G小内存服务器
- AI开发项目的API代理
- 多个Gemini API密钥的负载均衡
- 需要高可用性的API服务
- 学习Go语言网络编程

## ⚙️ 配置说明

### config.json 配置文件
```json
{
  "port": 8000,
  "backends": [
    {
      "url": "https://generativelanguage.googleapis.com",
      "api_key": "你的第一个API密钥"
    },
    {
      "url": "https://generativelanguage.googleapis.com", 
      "api_key": "你的第二个API密钥"
    }
  ],
  "health_check": {
    "enabled": true,
    "interval_seconds": 30,
    "timeout_seconds": 5,
    "path": "/v1beta/models"
  },
  "max_failures": 3,
  "log_level": "info"
}
```

### 配置参数说明

| 参数 | 说明 | 默认值 | 可选值 |
|------|------|--------|--------|
| `port` | 监听端口 | 8000 | 1-65535 |
| `backends[].url` | 后端API地址 | - | Gemini API URL |
| `backends[].api_key` | API密钥 | - | 你的Gemini API Key |
| `health_check.enabled` | 启用健康检查 | true | true/false |
| `health_check.interval_seconds` | 检查间隔(秒) | 30 | 10-3600 |
| `health_check.timeout_seconds` | 超时时间(秒) | 5 | 1-30 |
| `health_check.path` | 检查路径 | "/v1beta/models" | API路径 |
| `max_failures` | 最大失败次数 | 3 | 1-10 |
| `log_level` | 日志级别 | "info" | debug/info/warning/error |

## 🚀 快速开始

### 1. 环境准备

#### 安装Go语言
```bash
# Windows
# 访问 https://golang.org/dl/
# 下载 go1.21.x.windows-amd64.msi 并安装

# Linux
wget https://go.dev/dl/go1.21.0.linux-amd64.tar.gz
sudo tar -C /usr/local -xzf go1.21.0.linux-amd64.tar.gz
export PATH=$PATH:/usr/local/go/bin

# macOS
brew install go

# 验证安装
go version
```

### 2. 一键安装

#### Linux/Mac
```bash
chmod +x install.sh
./install.sh
```

#### Windows
```cmd
install.bat
```

### 3. 配置API密钥

编辑 `config.json` 文件，替换示例密钥为你的真实Gemini API密钥：

```json
{
  "backends": [
    {
      "url": "https://generativelanguage.googleapis.com",
      "api_key": "AIzaSyC_your_first_real_api_key_here"
    },
    {
      "url": "https://generativelanguage.googleapis.com", 
      "api_key": "AIzaSyC_your_second_real_api_key_here"
    }
  ]
}
```

### 4. 启动服务

#### 开发模式（直接运行）
```bash
go run main.go
```

#### 生产模式（编译后运行）
```bash
# 编译
go build -o gemini-proxy main.go

# 运行
./gemini-proxy        # Linux/Mac
gemini-proxy.exe      # Windows
```

## 🐳 部署方案

### 1. 直接部署

#### 后台运行
```bash
# Linux/Mac
nohup ./gemini-proxy > proxy.log 2>&1 &

# Windows
start /b gemini-proxy.exe > proxy.log 2>&1
```

#### Systemd服务（Linux）
创建 `/etc/systemd/system/gemini-proxy.service`：
```ini
[Unit]
Description=Gemini Proxy Load Balancer
After=network.target

[Service]
Type=simple
User=nobody
WorkingDirectory=/opt/gemini-proxy
ExecStart=/opt/gemini-proxy/gemini-proxy
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

启动服务：
```bash
sudo systemctl enable gemini-proxy
sudo systemctl start gemini-proxy
sudo systemctl status gemini-proxy
```

### 2. Docker部署

#### Dockerfile
```dockerfile
FROM golang:1.21-alpine AS builder
WORKDIR /app
COPY . .
RUN go build -o gemini-proxy main.go

FROM alpine:latest
RUN apk --no-cache add ca-certificates
WORKDIR /root/
COPY --from=builder /app/gemini-proxy .
COPY --from=builder /app/config.json .
EXPOSE 8000
CMD ["./gemini-proxy"]
```

#### 构建和运行
```bash
# 构建镜像
docker build -t gemini-proxy .

# 运行容器
docker run -d \
  --name gemini-proxy \
  -p 8000:8000 \
  -v $(pwd)/config.json:/root/config.json \
  gemini-proxy

# 查看日志
docker logs -f gemini-proxy
```

## 🔧 工作原理

### 负载均衡算法
```
请求1 -> API密钥1
请求2 -> API密钥2
请求3 -> API密钥1
请求4 -> API密钥2
...
```

### 健康检查机制
```
每30秒检查 -> GET /v1beta/models?key=xxx
HTTP 200 -> 标记为健康
HTTP 4xx/5xx -> 失败计数+1
失败次数 >= 3 -> 标记为不健康
恢复 HTTP 200 -> 重置为健康
```

### 故障转移逻辑
```
选择下一个后端 -> 检查是否健康
健康 -> 使用该后端
不健康 -> 跳过，选择下一个
所有都不健康 -> 使用第一个作为fallback
```
